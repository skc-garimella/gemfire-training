Environment Settings

Java Caching API Programming Example

Access the GemFire Release Notes and the GemFire User's Guide at the Pivotal GemFire Documentation Page.

Before running the programming examples, make sure your environment is set according to the specifications listed here.

The general environment settings referred to below are documented in Installing Pivotal GemFire section of the GemFire User's Guide. The commands and settings listed are generally Unix-centric, so adjust as necessary for other operating systems.

  1. Temporarily disable your firewall while you are running the examples. This ensures that the examples will perform as expected.

  2. Change directory to the top-level GemFire product directory. If you are in the top-level product directory, you should see a number of subdirectories, including bin, defaultConfigs, docs, lib, and DiskConverter.

  3. If you have not already done so, install Java. You must have a GemFire-compatible JDK installed on your system. See Pivotal GemFire Supported Configurations for a list of supported Java versions. You can download the appropriate JDK from http://www.oracle.com/technetwork/java/index.html.

    Note: A JDK is required for running gfsh commands, which are used in some of these examples.

  4. Set the environment variable JAVA_HOME to point to your Java JDK installation. For example:

    Linux, MacOSX, and Solaris Bourne and Korn shells (sh, ksh, bash)

    JAVA_HOME=yourJDK;export JAVA_HOME

    Windows®

    set JAVA_HOME=yourJDK

  5. Set the GF_JAVA environment variable to point to the java executable in your Java JDK environment. The GF_JAVA variable is used by GemFire scripts.

    6. These are example GF_JAVA configurations. In these examples, yourJRE or yourJDK points to the top-level JRE or JDK directory.

    Solaris®, Linux®, and AIX® Bourne and Korn shells (sh, ksh, bash)

    If just the JRE runtime is installed: GF_JAVA=yourJRE/bin/java;export GF_JAVA

    If the full Java JDK is installed: GF_JAVA=yourJDK/jre/bin/java;export GF_JAVA

    Windows®

    If just the JRE runtime is installed: set GF_JAVA=yourJRE/bin/java.exe

    If the full Java JDK is installed: set GF_JAVA=yourJDK/jre/bin/java.exe

  6. Add <Path to GemFire product dir>/bin to your PATH.

  7. Add <Path to GemFire product dir>/examples/dist/classes to your CLASSPATH.
    For example, this could be C:\GemFireXX/examples/dist/classes, where GemFireXX is the GemFire top-level product directory.

  8. Add <Path to GemFire product dir>/lib/gemfire.jar to the CLASSPATH.

  9. Add <Path to GemFire product dir>/lib/antlr.jar to the CLASSPATH.

  10. If you are running the security exercises in the clientServerConfigs example, include this setting:
    Add <Path to GemFire product dir>/lib/gfSecurityImpl.jar to CLASSPATH.
